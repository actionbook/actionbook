# Growth Playbook for Actionbook

A community-contributed guide to help Actionbook reach more developers who need reliable browser automation for AI agents.

## Positioning

Actionbook occupies a unique space: **the infrastructure layer between AI agents and the web**. While tools like Playwright handle browser automation and AI agents handle reasoning, Actionbook bridges the gap with pre-computed action manuals that eliminate the need for agents to parse raw DOM.

**Core differentiators to emphasize:**
- 10× faster execution (no HTML parsing)
- 100× token savings (semantic JSON vs full DOM)
- Resilient to website changes (manuals update, not agent code)
- Framework-agnostic (works with any LLM/agent framework)

## Content Strategy

### Blog posts that work for this project:
1. **"How Actionbook reduces AI agent token costs by 100×"** - Technical deep-dive with benchmarks
2. **"Building a flight booking agent in 50 lines"** - Tutorial format
3. **"Why we built action manuals instead of fine-tuning"** - Founder story / architecture decision
4. **"Actionbook vs Browser-use vs Raw Playwright: Honest comparison"** - Competitive analysis (shows confidence)
